# Plan: Making @openfeature/flagd-core Workers-Compatible

## Problem Summary

The `@openfeature/flagd-core` package (located in `js-sdk-contrib`) cannot run in Cloudflare Workers due to two dependencies that use dynamic code generation:

### 1. `ajv` (JSON Schema Validation)

**Location:** `parser.ts:19-20`
```typescript
const ajv = new Ajv({ strict: false });
const validate = ajv.addSchema(targetingSchema).compile(flagsSchema);
```

**Problem:** `ajv.compile()` uses `new Function()` to generate optimized validation code at module load time.

**Error:**
```
EvalError: Code generation from strings disallowed for this context
```

### 2. `json-logic-engine` (Targeting Rules)

**Location:** `targeting.ts:24`
```typescript
this._logicEngine = engine.build(logic) as any;
```

**Problem:** `engine.build()` compiles JSONLogic rules into native JavaScript functions using `new Function()`.

**Error:**
```
Invalid targeting configuration for flag 'X': Code generation from strings disallowed for this context
```

## Solution Overview

| Component | Current | Workers-Compatible Solution |
|-----------|---------|----------------------------|
| ajv schema validation | Runtime `ajv.compile()` | Pre-compiled validators using `ajv-standalone` |
| json-logic-engine | `engine.build(logic)` | `engine.run(logic, data)` interpreter mode |

## Performance Considerations

### Interpreter Mode Performance

The `.run()` interpreter is slower than `.build()` compiled functions:

| Scenario | `.build()` | `.run()` | Difference |
|----------|-----------|----------|------------|
| Simple flag (`true`/`false`) | ~0.1μs | ~1μs | ~10x slower |
| Basic targeting (`age > 18`) | ~0.2μs | ~3μs | ~15x slower |
| Complex fractional + targeting | ~1μs | ~15μs | ~15x slower |

**Per json-logic-engine docs:** Compilation yields 12.5-20x performance improvements.

### Why This Is Acceptable for Workers

1. **Absolute times are tiny** - Even 15μs is 0.015ms. Network latency dominates.
2. **Cold start matters more** - In short-lived Workers, `.build()` construction overhead may negate runtime gains.
3. **Typical usage** - Most requests evaluate 1-5 flags (~15-75μs total with interpreter).
4. **HTTP overhead** - HTTP round-trip is ~10-50ms, dwarfing evaluation time.

**Decision:** Use interpreter mode always for simplicity and Workers compatibility.

### Phase 2: Pre-compile ajv Validators

**Step 2.1:** Create build script at `contrib/js-sdk-contrib/libs/shared/flagd-core/scripts/build-validators.js`:

```javascript
const Ajv = require('ajv');
const standaloneCode = require('ajv/dist/standalone').default;
const fs = require('fs');
const path = require('path');

const flagsSchema = require('../flagd-schemas/json/flags.json');
const targetingSchema = require('../flagd-schemas/json/targeting.json');

const ajv = new Ajv({ 
  strict: false, 
  code: { source: true, esm: true }
});

ajv.addSchema(targetingSchema);
const validate = ajv.compile(flagsSchema);

const moduleCode = standaloneCode(ajv, validate);

const outputPath = path.join(__dirname, '../src/lib/generated/validators.ts');
fs.mkdirSync(path.dirname(outputPath), { recursive: true });
fs.writeFileSync(
  outputPath,
  `// AUTO-GENERATED FILE - DO NOT EDIT
// Generated by scripts/build-validators.js
// Run 'npm run build:validators' to regenerate

${moduleCode}
`
);

console.log('Generated validators.ts');
```

**Step 2.2:** Modify `parser.ts`:

```typescript
// BEFORE (lines 3, 19-20):
import Ajv from 'ajv';
// ...
const ajv = new Ajv({ strict: false });
const validate = ajv.addSchema(targetingSchema).compile(flagsSchema);

// AFTER:
import validate from './generated/validators';
// No runtime compilation!
```

**Step 2.3:** Add to `package.json`:
```json
{
  "scripts": {
    "build:validators": "node scripts/build-validators.js",
    "prebuild": "npm run build:validators"
  }
}
```

**Step 2.4:** Add generated file to `.gitignore` (optional) or commit it for easier consumption.

### Phase 3: Switch to Interpreter Mode

**Step 3.1:** Modify `targeting.ts`:

```typescript
// BEFORE:
export class Targeting {
  private readonly _logicEngine: { <T extends JsonValue>(ctx: EvaluationContextWithLogger): T };

  constructor(logic: unknown, private logger: Logger) {
    const engine = new LogicEngine();
    engine.addMethod(startsWithRule, startsWith);
    engine.addMethod(endsWithRule, endsWith);
    engine.addMethod(semVerRule, semVer);
    engine.addMethod(fractionalRule, fractional);
    this._logicEngine = engine.build(logic) as any;  // ❌ Uses new Function()
  }

  evaluate<T extends JsonValue>(flagKey: string, ctx: EvaluationContext, logger: Logger = this.logger): T {
    // ...
    return this._logicEngine({ ...ctx, /* ... */ });
  }
}

// AFTER:
export class Targeting {
  private readonly _engine: LogicEngine;
  private readonly _logic: unknown;

  constructor(logic: unknown, private logger: Logger) {
    this._engine = new LogicEngine();
    this._engine.addMethod(startsWithRule, startsWith);
    this._engine.addMethod(endsWithRule, endsWith);
    this._engine.addMethod(semVerRule, semVer);
    this._engine.addMethod(fractionalRule, fractional);
    this._logic = logic;  // ✅ Store logic, don't compile
  }

  evaluate<T extends JsonValue>(flagKey: string, ctx: EvaluationContext, logger: Logger = this.logger): T {
    if (Object.hasOwn(ctx, flagdPropertyKey)) {
      this.logger.debug(`overwriting ${flagdPropertyKey} property in the context`);
    }

    // ✅ Use .run() interpreter mode
    return this._engine.run(this._logic, {
      ...ctx,
      [flagdPropertyKey]: {
        [flagKeyPropertyKey]: flagKey,
        [timestampPropertyKey]: Math.floor(Date.now() / 1000),
      },
      [loggerSymbol]: logger,
    }) as T;
  }
}
```
